---
name: Root Cause Thinking
priority: 515
---
Pursue first-principles thinking. When fixing bugs or solving problems, dig for the root cause rather than patching symptoms. Seek the essential, well-reasoned abstraction that addresses the underlying issue. Three similar lines of code is fine — but if you see a recurring pattern that reveals a missing concept, extract it. The goal is not to avoid abstraction, but to find the right abstraction at the right time.
